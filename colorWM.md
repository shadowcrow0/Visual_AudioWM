# Color Working Memory / Perceptual Separability 筆記

## Q1: perceptual separability 建構在工作記憶作業中，涉及跨 modality 整合，兩個顏色相近時人們能否在知覺上把它們分開處理？

這牽涉到兩個不同層次的「分開處理」，在 VAWM 設計裡建議要分開來看：

**1. GRT 的 perceptual separability 本來是設計給「同時呈現、無記憶延遲」的辨識作業用的**（Ashby & Townsend, 1986），問的是兩個物理維度在單次知覺當下會不會互相干擾。它沒有內建「延遲後、跨通道整合」這件事，所以不能直接拿來回答工作記憶裡的問題——工作記憶要問的其實是另一件事：**兩個顏色在編碼當下可能可以被知覺區分,但經過保留期(retention interval)、又要同時處理聽覺訊息之後,這個區分還能不能維持住,不會被弄混。** 這是記憶維持/提取階段的問題，不是知覺辨識階段的問題,兩者常常不一致。

**2. 工作記憶文獻對這個問題已經有相對明確的答案：能分開辨識 ≠ 記憶中會保持分開。**
- 連續回報式顏色工作記憶研究(如 Zhang & Luck, 2008; Bays, Catalao & Husain, 2009 的 swap-error/misbinding 模型)發現，當多個項目的顏色彼此距離越近，受試者事後越容易把 A 的顏色錯配到 B 身上(swap error)，而且這個機率是「顏色距離」的連續函數,不是有/無的二分——也就是說,即使兩色在直接比較時可以分辨(discriminable),放進多項目工作記憶、延遲一段時間後,相近的顏色仍然會顯著提高誤配率、降低回憶精確度(precision 下降、雜訊上升),這是族群編碼(population coding)重疊造成的干擾,不是「知覺分不開」,而是「記憶表徵擠在一起、容易漂移混淆」。
- 這個效應在有跨通道整合(顏色要跟聲音綁定成同一個項目)時通常會更明顯：工作記憶要同時維持「這個顏色屬於哪個聲音」的綁定關係(binding),相近顏色本身的記憶雜訊,加上跨通道綁定要額外負荷注意力資源,兩者疊加會讓相近顏色配對的錯誤率比純視覺、單一維度時更高。也就是說,雙通道整合通常是讓相似性效應惡化,而不是緩解。

**對設計的實際意涵**：用近 JND(ΔE00~2)的顏色當作 VAWM 的視覺特徵，預期在有延遲、有跨通道整合(聲音)的情況下，混淆/誤配率會比單純知覺辨識作業(無延遲、直接比較)高出不少——這其實是可以拿來驗證的假設，而不是設計上的缺陷：可以同時收「知覺辨識基準線」(無延遲、直接比較兩色，測 d')跟「工作記憶+跨通道」條件(VAWM 作業)，比較同一組顏色在兩種作業下的錯誤率差異，這個差異量本身就能回答「知覺分得開的東西，記憶裡還分不分得開」。

## Q2: 針對候選色票 set 13,14,15,17,18 的建議

從挑的這幾組（13,14,15 落在 h=293°；17,18 落在 h=303°），可以看出一個重點：已經避開了邊界列（h≈283°，也就是 set 9–12），這個選擇本身在文獻上是對的方向，理由如下：

**為什麼不用邊界色相(283°)：跨類別配對反而更容易被語言破解**
範疇知覺(categorical perception)文獻的核心發現是——**跨類別**(一個偏藍命名、一個偏紫命名)的配對，即使物理色差(ΔE)跟同類別配對一樣小，也會因為兩者「有不同的顏色名字」而變得更容易區分/命名(這正是 CP 效應的定義:類別邊界會放大跨界差異的知覺)。如果選在邊界上(283°)取一對相近色，很可能一個被叫「藍」、一個被叫「紫」，反而給了受試者一個現成的語言標籤去區分——這正好跟「口語分不出來」的目標互相矛盾。選在同一類別內(293°或303°，兩色都會被叫同一個名字，例如都叫「紫」)才符合原本的設計目的。選 293/303 而不是 283，方向是對的。

**293° vs 303°：建議用 303°，不是 293°**
看實際色票：293° 那一列(#787FC1 這種)藍色分量還是偏高，很可能有些受試者仍然會標成「藍紫」甚至直接叫「藍」，等於還是有一部分人會落在類別邊界的模糊區、給了可語言化的線索；303° 那一列(#887BBC 這種)已經明顯偏向紫/薰衣草色，比較不會有人叫它「藍」，同一類別標籤的一致性比較高，更能滿足「兩色都拿不到可區分的名字」這個前提。293° 離邊界(283°)只有 10°，303° 離邊界 20°，安全邊際也比較大——個體之間對邊界位置本來就有落差，303° 比較不會因為個別受試者邊界劃得不一樣而讓其中一色被視為跨類別。

**ΔE00 該用哪個：不要只挑一個,用漸層(2/4/6)**
根據 Bays et al. (2009) 的 swap-error 模型，記憶中的誤配率是顏色距離的**連續函數**，不是有無效應的二分——只測一個 ΔE 沒辦法確定「誤配率隨相似度上升」這條劑量反應曲線，至少要 2–3 個色差層級才能看出趨勢，並排除「隨便有差異就會混淆」這種天花板/地板效應的替代解釋。

**具體建議**：用 **set 17、18、19**（都是 h=303°，ΔE00 = 2 / 4 / 6 的完整漸層組），而不是混用 293° 跟 303°。set 19 雖然沒被列出來，但已經在同一份 CSV(`bv_candidates_for_advisor.csv`)裡了（`#827CBE` / `#9377B7`，ΔE00=6.01），三組色相位置一致，只有色差在變，是乾淨的單一自變項操弄。293° 那組(13/14/15)可以留著當作「同類別、但更接近邊界」的第二個色相位置，之後如果想測「離邊界距離」是否也影響誤配率，可以拿來做對照，但不建議當主要操弄組。

## Q3: 「邊界距離」是什麼意思 + 文獻 APA 引用

**「邊界距離」的定義：**
指的是選的顏色，在色相軸上距離「藍/紫命名分界點」有多遠。用 283° 當作分界點的粗略估計(藍紫兩個顏色名字切換的地方)，293° 離它 10°，303° 離它 20°。距離分界點越近，個體之間對「這算藍還是算紫」的判定越容易出現分歧(有人覺得還是藍、有人覺得已經是紫)，也就越可能被賦予不同語言標籤，讓配對變得可語言化；離分界點越遠、越靠近某個類別的典型色(prototype)，大家對命名的共識越高，越不會被語言標籤破解。

**within-category color working memory 對應的文獻：Bae, Olkkonen, Allred, & Flombaum (2015)**
這篇完全針對這個問題：他們發現顏色在工作記憶中的表徵是「類別(category)」跟「特定值(particular)」兩者結合的產物，**越靠近類別邊界的顏色，記憶時的偏誤(bias)越大、精確度越受類別拉扯影響；越靠近類別典型色(prototype)的顏色，記憶偏誤越小、越穩定**。這篇直接支持用離邊界較遠的 303°（更接近「紫」的典型色)而不是 293°（較靠近邊界)的理由——不只是命名一致性的問題，連記憶精確度本身都會受邊界距離影響。
